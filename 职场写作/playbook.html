◾ 怎么用这一页

六个高频场景，每个给一个能填空就用的模板

下面每个模板都是把五个原则套进具体场景的结果。黄色是你要替换的槽位。不用逐字照抄，抄的是结构和顺序——尤其是"什么放最前面"。

◾ 场景 01 · 向上请示 / 要资源

要东西：先说要什么，再说为什么值得给

【请求】想申请2 名外包 / 5 万预算 / 延期 1 周，用于X 项目。 【为什么需要】一句话说清缺口：现在的资源撑不到什么程度 【能换来什么】给了之后的具体产出 / 时间节点 【不给的后果】诚实但不威胁：会导致什么，可接受的替代是什么 【需要您】在 X 前拍板 / 我默认周五推进。
要点：领导批不批，看的是"投入换产出划不划算"，不是你多辛苦。所以把"要什么 + 换来什么"放前面，别用一大段过程叙述铺垫。给一个默认选项（"不回复我就按 X 推进"）能大幅提高响应率。
◾ 场景 02 · 周报 / 工作汇报

周报不是流水账，是"结果 + 风险 + 需求"

【本周关键结果】1–3 条，带数字：做完了什么、达到什么指标 【进展中】在推进、还没出结果的事 + 预计完成时间 【风险 / 卡点】哪里可能出问题、需要谁帮忙——这栏最有价值 【下周计划】2–3 条重点，不是任务清单全录
✓ 写结果"完成支付模块联调，成功率从 92% 提到 99.5%"
✕ 写动作"本周开了 3 个会，跟进了支付模块，写了文档"

领导从周报里最想看的其实是"风险/卡点"那一栏——它让他能提前介入。很多人却把它省了，只报喜。把卡点写清楚，反而显得你 owner 意识强。

◾ 场景 03 · 汇报一个坏消息

坏消息别埋，坏消息 + 已做的动作 + 需要的支持

出事了不敢直说、把坏消息裹在一堆解释里慢慢引出——这是职场写作最常见的错误，因为它让领导觉得你在遮掩。正确做法：开门见山说问题，紧接着说你已经做了什么止损，最后说需要什么支持。顺序本身就在传递"我在扛事，不是甩锅"。

【发生了什么】一句话讲清坏消息：影响范围 + 严重程度 【我已经做了】已采取的止损动作——证明你在扛 【还需要】需要的决策 / 资源 / 授权，越具体越好 【后续】下一个同步节点：什么时候再更新
✕ 遮掩式

领导，这周确实遇到了一些挑战，主要是因为一些客观原因，我们也一直在积极想办法，目前还在努力协调中，可能进度上会稍微有一点点影响……

✓ 扛事式

领导，A 客户上线延期，预计推迟 3 天。我已协调测试团队今晚加班，并向客户发了说明、争取到他们同意。需要您帮忙：批准两天加班预算。周三 10:00 我再同步进度。

◾ 场景 04 · 写方案 / 提案

方案用金字塔：先给建议，再给论证

【背景】一句话：现状是什么、为什么现在要解决（SCQA 的 S+C） 【建议】我建议采用方案 A（把结论放在最前，别让人读到最后才知道你推荐哪个） 【为什么是 A】 · 对比维度一（成本）：A vs B vs C · 对比维度二（风险）：A vs B vs C · 对比维度三（时间）：A vs B vs C 【怎么落地】关键步骤 + 里程碑 + 谁负责 【需要决策】这次要拍板的具体事项
要点：很多人写方案喜欢"客观中立地罗列 A/B/C 三个方案，最后让领导选"——这是把该你做的判断推给了领导。先亮出你的推荐，再用对比论证它，反而更专业。留一个"我推荐 A，但如果更看重成本可以选 B"的判断，比不表态更有价值。
◾ 场景 05 · 绩效 / 反馈沟通

给反馈用 SBI：具体行为 + 影响 + 建议

反馈最忌讳笼统评价（"你最近状态不太好""你要更主动一点"）——对方不知道具体指什么，只感到被否定。用 SBI：Situation（什么场合）→ Behavior（具体做了/没做什么行为）→ Impact（造成了什么影响），最后给一条可执行的建议。

✕ 笼统评价

你最近工作态度有点问题，responsibility 不太够，希望你能更有主人翁意识一点。

✓ SBI 具体化

场景：上周三的需求评审会。行为：会上没提出你负责模块的两个已知风险，会后才在群里说。影响：导致方案返工，多花了一天。建议：下次评审前把风险点列成清单，会上主动抛出来。

⚠ 这类内容别让 AI 代写后原封发出。研究显示，当员工发现主管用 AI 撰写这种关系性、评价性沟通时，对其真诚度的评分会从 83% 跌到 40–52%（Cardon & Coman, 2025）。表扬、批评、道歉、激励——这些"人对人"的话，AI 可以帮你理思路，但最后一定要用自己的话、带具体细节写出来。详见用 AI 不翻车。
◾ 场景 06 · 拒绝 / 说不

拒绝：先接住，再给理由，最后给替代

【接住】先肯定对方的诉求合理 / 理解处境 【说不 + 为什么】明确拒绝 + 一个真实的、不卑不亢的理由 【给替代】我没法做 X，但可以做 Y / 建议找 Z / 下个周期再排
✕ 硬邦邦 or 含糊

这个我们做不了。 ——（太硬，像甩脸）
嗯……我看看吧，可能有点难，尽量……——（含糊，对方以为还有戏）

✓ 接住 + 替代

理解你们这个需求很急。这个排期我们这个 sprint 确实接不了，手上 A 项目占满了。替代方案：如果只做核心那部分，我可以下周挤 2 天；或者下个 sprint 排在最前。你看哪个行？
